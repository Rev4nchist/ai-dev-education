Integrating MCP with Development Tools

Step-by-step guides for integrating Model Context Protocol with Windsurf and Cursor

Getting Started with MCP Integration

The Model Context Protocol (MCP) can be integrated with various development tools to enhance your AI-assisted workflow. In this guide, we'll focus on integrating MCP with two popular tools: Windsurf's Cascade chatbot and Cursor IDE.

Both tools offer powerful MCP integration capabilities, but they have slightly different approaches and configurations. We'll walk through the step-by-step process for each tool, allowing you to choose the one that best fits your development workflow.

Integrating MCP with Windsurf's Cascade

Windsurf is Codeium's next-generation AI IDE, featuring Cascade, an agentic chatbot that supports Model Context Protocol (MCP) for interfacing with external tools. This guide will walk you through the process of setting up and using MCP with Windsurf's Cascade.

1

Prerequisites

Before you begin, ensure you have the following:

  • Codeium Windsurf Editor: Download and install Windsurf from codeium.com/windsurf.
  • Node.js (>= v18) and npm: Ensure Node.js (version 18 or later) and npm are installed. Download them from nodejs.org.
  • API Keys: Depending on the MCP servers you plan to use, you may need specific API keys (e.g., Neon API key for database access).
2

Accessing MCP Configuration in Windsurf

To configure MCP Servers in Windsurf, you need to access the MCP configuration file:

  1. Open Windsurf.
  2. Open Cascade by using âŒ˜L (Mac) or Ctrl+L (Windows/Linux).
  3. Find the toolbar above the Cascade input and click the hammer icon (ðŸ”¨).
  4. Click the "Configure" button to open the ~/.codeium/windsurf/mcp_config.json file in the IDE.

Note: The MCP configuration file is located at ~/.codeium/windsurf/mcp_config.json. This file contains the list of MCP servers that Cascade can connect to.

3

Configuring an MCP Server

In the mcp_config.json file, you need to specify a list of MCP servers. Here's a basic template for the configuration:

{
  "mcpServers": {
    "server-name": {
      "command": "command-to-run",
      "args": ["arg1", "arg2", "..."]
    }
  }
}

Let's look at a specific example using the Neon MCP Server for database access:

{
  "mcpServers": {
    "neon": {
      "command": "npx",
      "args": ["-y", "@neondatabase/mcp-server-neon", "start", "YOUR_NEON_API_KEY"]
    }
  }
}

Components of this configuration:

  • "neon": This is a name you choose for your MCP server connection.
  • "command": This is the command Windsurf will execute to start the MCP server.
  • "args": These are the arguments passed to the command.

Security Warning: Be careful with API keys in your configuration file. Never share your configuration file publicly or commit it to a public repository.

4

Activating the MCP Server

After configuring your MCP server:

  1. Save the mcp_config.json file.
  2. Click "Refresh" (ðŸ”„) in the MCP toolbar in Windsurf Cascade.
  3. If the integration is successful, the toolbar should indicate that you have MCP servers available, and you should see "1 available MCP server" (or more if you configured additional servers).
5

Using MCP with Cascade

Once your MCP server is configured and active, you can use it with Cascade:

  1. Open Cascade using âŒ˜L (Mac) or Ctrl+L (Windows/Linux).
  2. Ask Cascade to perform tasks that require the MCP server. For example, if you've configured the Neon MCP server, you might ask: "List all tables in my Neon database."
  3. Cascade will automatically use the appropriate MCP server to fulfill your request.

Note: Be aware that Cascade currently executes commands directly from your prompts without confirmation, unlike some other IDE's and apps. Review your requests thoroughly to avoid unintended or unwanted actions.

Integrating MCP with Cursor

Cursor is an intelligent IDE that implements an MCP client supporting an arbitrary number of MCP servers. This guide will walk you through the process of setting up and using MCP with Cursor.

1

Prerequisites

Before you begin, ensure you have the following:

  • Cursor IDE: Download and install Cursor from cursor.com.
  • Node.js and npm: Ensure Node.js and npm are installed for running MCP servers.
  • API Keys: Depending on the MCP servers you plan to use, you may need specific API keys.
2

Accessing MCP Settings in Cursor

To configure MCP Servers in Cursor:

  1. Open Cursor.
  2. Go to Cursor Settings > Features > MCP.
  3. Click on the "+ Add New MCP Server" button.

Note: Cursor supports both stdio and sse transports for MCP servers, giving you flexibility in how you connect to your servers.

3

Configuring an MCP Server

When adding a new MCP server, you'll need to fill out a form with the following information:

  1. Select the transport type under Type (stdio or sse).
  2. Fill out a nickname for the server (Name).
  3. Provide either the command to run or the URL of the server, depending on the transport type.

For example, to configure a weather server using stdio transport:

  • Type: stdio
  • Name: Weather Server
  • Command: node ~/mcp-quickstart/weather-server-typescript/build/index.js

Or to configure a server using sse transport:

  • Type: sse
  • Name: Sample Tool
  • URL: http://localhost:8765
4

Project-specific MCP Configuration

You can also configure project-specific MCP servers using .cursor/mcp.json. The file follows this format:

// For stdio transport
{
  "servers": [
    {
      "name": "My Server",
      "transport": {
        "kind": "stdio",
        "command": "node /path/to/server.js"
      }
    }
  ]
}

// For sse transport
{
  "servers": [
    {
      "name": "My Server",
      "transport": {
        "kind": "sse",
        "url": "http://localhost:8765"
      }
    }
  ]
}

Tip: Project-specific configurations are useful when different projects require different MCP servers or configurations.

5

Using MCP Tools in Cursor

Once your MCP server is configured:

  1. The Composer Agent will automatically use any MCP tools that are listed under Available Tools on the MCP settings page if it determines them to be relevant.
  2. To prompt tool usage intentionally, simply tell the agent to use the tool, referring to it either by name or by description.

Tool Approval: By default, when Agent wants to use an MCP tool, it will display a message asking for your approval. You can expand the message to see the tool call arguments.

Yolo Mode: You can enable Yolo mode to allow Agent to automatically run MCP tools without requiring approval, similar to how terminal commands are executed. This can be enabled in Cursor settings.

Building a Custom MCP Server for Cursor

You can also build your own custom MCP server for Cursor. Here's a basic example of creating a simple Jira integration:

1

Project Setup

mkdir cursor-jira-tool  
cd cursor-jira-tool  
npm init -y  
npm install @modelcontextprotocol/sdk jira-client typescript  
npm install --save-dev @types/node
2

Configure TypeScript

// tsconfig.json
{  
  "compilerOptions": {  
    "target": "ES2020",  
    "module": "ES2020",  
    "moduleResolution": "node",  
    "outDir": "./dist",  
    "strict": true  
  }  
}
3

Implement MCP Server

// src/index.ts
import { McpServer } from '@modelcontextprotocol/sdk/server/mcp';  
import { z } from 'zod';  
  
const server = new McpServer({  
  name: "jira-tools",  
  version: "1.0.0"  
});  

// Define commands  
server.tool("add-comment", "Add a comment to a Jira ticket", {  
  ticket: z.string(),  
  comment: z.string()  
}, async ({ ticket, comment }) => {  
  // Implementation logic here
  return { success: true, message: `Added comment to ${ticket}` };  
});  

server.tool("update-ticket", "Update ticket status", {  
  ticket: z.string(),  
  status: z.string()  
}, async ({ ticket, status }) => {  
  // Implementation logic here
  return { success: true, message: `Updated ${ticket} status to ${status}` };  
});

// Start the server
server.start();
4

Build and Configure

  1. Build your project: npm run build
  2. In Cursor, go to Features > MCP
  3. Add a new MCP server pointing to your built dist/index.js file

Comparing MCP Integration: Windsurf vs. Cursor

Both Windsurf and Cursor offer powerful MCP integration capabilities, but they have different approaches and features. Here's a comparison to help you choose the right tool for your needs:

Feature Windsurf (Cascade) Cursor
MCP Transport Support stdio only stdio and sse
Configuration Location ~/.codeium/windsurf/mcp_config.json Cursor Settings UI or .cursor/mcp.json
Tool Approval No approval required (executes directly) Requires approval by default (Yolo mode optional)
Project-Specific Config Limited support Full support via .cursor/mcp.json
UI for Configuration Basic (requires editing JSON) More comprehensive UI
Integration Style Chatbot-focused (Cascade) Agent and Composer integration
Base Editor Codeium's custom editor Built on VS Code

When to Choose Windsurf

  • If you prefer a more conversational, chatbot-driven approach to AI assistance
  • If you're already using other Codeium tools and want tight integration
  • If you want direct execution of MCP commands without approval steps

When to Choose Cursor

  • If you need support for both stdio and sse transports
  • If you prefer a VS Code-like environment
  • If you want more control over tool approval and execution
  • If you need project-specific MCP configurations

Tip: Both tools are actively developing their MCP integration capabilities, so it's worth checking their latest documentation for updates and new features.

Common MCP Integration Challenges

When integrating MCP with development tools, you might encounter some common challenges:

API Key Management

Challenge: Securely storing and managing API keys for various MCP servers.

Solution: Use environment variables or secure credential storage solutions rather than hardcoding keys in configuration files.

Server Compatibility

Challenge: Ensuring MCP servers are compatible with your chosen development tool.

Solution: Check the documentation for both the MCP server and the development tool to confirm compatibility, particularly regarding transport types.

Debugging MCP Issues

Challenge: Identifying and resolving issues with MCP server connections.

Solution: Check server logs, verify network connectivity, and ensure the server is running correctly. Both Windsurf and Cursor provide ways to view MCP server status.
